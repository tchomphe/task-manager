# Implementation Roadmap

## Overview

Eight sequential phases. Each phase has a single clear goal, explicit file-level deliverables, and a Definition of Done that must be fully satisfied before the next phase begins. Phases 3 and 4 can be parallelized once Phase 2 is complete (the DTO contract is stable at that point).

---

## Global Constraints

These rules apply in **every phase** without exception:

1. **DTOs only in API responses** — no EF entity type ever appears in an HTTP response body
2. **`userId` from JWT only** — always extracted from `User.FindFirstValue(ClaimTypes.NameIdentifier)`, never from the request body
3. **Every task/tag query scoped to current user** — `.Where(t => t.UserId == currentUserId)` is the first filter on every query
4. **All datetimes UTC** — stored and returned as UTC ISO 8601; clients handle timezone display
5. **No secrets committed** — `*.db`, `bin/`, `obj/`, `node_modules/`, `.env*` excluded via `.gitignore`

### Key Entity Rules

- `TaskItem.UserId` and `Tag.UserId` are non-nullable — never null in the DB
- `CreatedAt` and `UpdatedAt` default to `DateTime.UtcNow` in entity constructors
- `DueDate` nullable, stored as UTC; EF SQLite value converter ensures UTC kind is preserved on read
- No soft delete — tasks and tags are hard-deleted

### Definition of Done

- `dotnet ef database update` creates `taskmanager.db` with all tables
- `AppDbContext` compiles; entity relationships visible in migration snapshot

                                    # PUT    /api/tags/{id}   → 200 TagResponse (PATCH semantics)
                                    # DELETE /api/tags/{id}   → 204 No Content

backend/TaskManager.Api/

                                    # ITaskService, ITagService with DI
```

### Query Logic (in TaskRepository)

1. `.Where(t => t.UserId == userId)` — always first
2. Optional status filter, priority filter
3. Optional `.Where(t => t.Title.Contains(search) || t.Description.Contains(search))`
4. `.OrderByDescending(t => t.CreatedAt)` — stable default sort
5. `CountAsync()` for total (before pagination)
6. `.Skip((page-1) * pageSize).Take(pageSize)`

### Tag Handling

- Tags must be created explicitly via `POST /api/tags` before being referenced on tasks
- Task create/update accepts `string[]` tag names; `TagRepository.GetManyByNamesAsync` resolves them — throws 400 if any name not found for the user
- **PATCH semantics on update**: omit `Tags` → unchanged; `Tags: []` → cleared; `Tags: ["x"]` → replaced wholesale
- `TaskResponse.Tags` returns full `TagResponse[]` (id, name, color)
- `GET /api/tags` returns `TagResponse[]` for the current user — used for autocomplete/picker

### Definition of Done

- Full CRUD cycle works via Swagger or `api.http`
- Full CRUD for tags: `POST`, `PUT`, `DELETE` all enforce ownership; duplicate name returns 400
- `GET /api/tags` returns tags (with color) for the authenticated user
- `DELETE /api/tags/{id}` removes tag from all associated tasks
- Creating a task with a non-existent tag name returns 400
- Task responses return full `TagResponse[]` (id, name, color)
- User A `GET /api/tasks/{userBTaskId}` returns `404` (ownership enforced by scoped query)
- `GET /api/tasks?search=foo&page=2&pageSize=5` returns correct slice with accurate `TotalCount`
- `PUT /api/tasks/{id}` with omitted `Tags` leaves tags unchanged; with `Tags: []` clears all tags
- `dotnet build` exits 0 with no warnings

  TagChip.tsx                       # Small pill for a tag name

```

### Checkmark Shortcut

Clicking the checkmark on `TaskCard` calls `useUpdateTask` with `{ status: 'Done' }`. If the task is already `Done`, clicking again sets `{ status: 'Todo' }`. No new field — just a status PATCH.

### Data State Rules (non-negotiable)

- `TaskListPage` loading → 5x `SkeletonCard`; error → `ErrorMessage` with retry; empty → `EmptyState` with "Create your first task" CTA
- `TaskDetailPanel` loading → full-width skeleton; 404 (task not found) → close panel + show error toast

### URL State

- `?search=`, `?status=`, `?priority=`, `?page=` all live in URL — shareable and survives refresh
- Search debounced 300ms before updating URL

### Definition of Done

- Full CRUD cycle works in the browser end-to-end
- Checkmark toggles status between Done and Todo with immediate feedback
- Search + filters narrow results; pagination navigates pages; all URL params survive refresh
- All three data states visible for task list and detail panel
- No Axios calls in page or component files — all data through hooks
- `tsc --noEmit` exits 0
